## What's built and committed

- **`LMQCodec`** class (`src/tomat/float_codec.py`): unsigned 1-token
  empirical quantizer, compatible with `PatchTokenizer`'s
  `encode_signed`/`decode_signed` API. Loads codec from .npz (local or
  gs://) via `LMQCodec.load()`.
- **Lloyd-Max fit script** (`scripts/fit_lmq_codec_modal.py`): streams
  all train-full Zarrs through a 1 M-fine-bin histogram, merges across
  workers, runs Lloyd-Max iteration, saves to GCS. `retries=5` on the
  worker function + `chunk_size=50` tunable for preemption resilience.
- **Tokenizer integration** (`scripts/tokenize_patches.py`): accepts
  `--density-codec lmq --lmq-path gs://…` and writes the codec path
  into meta.json.
- **Modal tokenize wrapper** (`scripts/tokenize_patches_modal.py`):
  threads `lmq_path` through `main`/`parallel` entry points; image now
  has `gcsfs` + `tomat-gcp-sa` secret so the in-container tokenize can
  load the codec from GCS.
- **Density-aware loss subclass** (`marin/qwen3_density.py`):
  `Qwen3DensityLMHeadModel` + `Qwen3DensityConfig`. Overrides
  `compute_next_token_loss` with the formulation we converged on:
  ```
  at each position t with target y_t:
      if y_t is a density token:
          P = softmax(logits[t])
          E[ρ] = Σ_v P[v] · decode_all[v]     (decode_all has PENALTY
                                                for non-density tokens)
          L_1 = |E[ρ] − ρ_true|
      else:
          CE = -log P(y_t)
  total = CE + λ · L_1, reduced over masked positions
  ```
- **Training script wiring** (`marin/train_tomat_tpu.py`): new env
  vars `TOMAT_DENSITY_L1_WEIGHT`, `TOMAT_DENSITY_L1_MODE` (`add` or
  `replace`), `TOMAT_DENSITY_PENALTY`, `TOMAT_LMQ_PATH`. When
  `L1_WEIGHT > 0`, swaps in `Qwen3DensityConfig` + loads LMQ codec +
  configures density loss via `configure_density_loss(...)`. Falls
  through to vanilla Qwen3 when weight=0.

## Known issues / risks

1. **Loss subclass untested on TPU.** The Haliax↔JAX dance in
   `density_aware_loss()` drops to raw `jnp` for the heavy math.
   Might break on a sharded-vocab mesh. Plan: try on v6e-8 (single
   host) first; if that works, multihost is likely fine too.
2. **Fit preemption.** Modal's preemptible CPUs are kicking workers
   mid-run. `retries=5` should handle it but adds wall-clock time.
   If fit hangs too long past ~1 hr, inspect `/tmp/fit-lmq.log` and
   consider killing + relaunching with `chunk_size=100` (balance
   between restart overhead and preemption resilience).
3. **Vocab size jumps 6,792 → 18,568** under 16k-bin LMQ (the larger
   density vocab dominates). Embedding cost goes from 3.5% → ~9% of
   208M params. Acceptable, not ideal. Could shrink to 8k-bin LMQ
   if we want more compute in the transformer layers.
4. **PENALTY calibration.** Default = 10 × max(recon_points). If the
   model leaks probability to non-density tokens early in training,
   the L_1 loss will spike to PENALTY-scale values and could
   destabilize. Watch for this in the first 50 steps; if spiky,
   reduce PENALTY to ~2 × max(recon).
5. **"replace" mode never tested.** If `add` works, try `replace` as
   a separate run later.
